Providers are tried top to bottom. On a rate-limit (429), that provider cools down for RATE_LIMIT_COOLDOWN seconds (default 60s) and the next one is tried automatically — no action needed on your end.

Set PROVIDER=auto to use all available providers, or PROVIDER=groq to prefer one with auto-fallback to the rest. Add API keys as env vars to unlock more providers. See awesome-free-llm-apis.

What can I help you build?

Auto-fallback across free LLM APIs. Hit a rate limit on one — it silently moves to the next.
